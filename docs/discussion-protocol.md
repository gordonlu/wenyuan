# 合议协议

## 席位

- 谋远席：发散思考、替代路线、长期机会和组合可能。
- 经世席：用户价值、现实路径、资源成本和最小落地方案。
- 持正席：事实、逻辑、隐藏假设、风险、原则和边界。

三席平等，不存在固定主持人或裁判。

## 阶段

1. 创建议题：用户提交标题、议题正文和补充背景。
2. 独议：三席并行执行，互相看不到其他席输出。
3. 批议：每席读取另外两席的结构化摘要并提出质询。
4. 复议：每席吸收批议并形成正式策案。
5. 阁议：三席匿名评价全部策案并投票。
6. 多数决：同一策案获得至少两个 final choice 即形成多数。
7. Convergence：若三票分散，只允许一次合案复议和重新投票。

## JSON 约束

所有模型阶段都必须返回结构化 JSON。解析失败时：

1. 保存原始输出。
2. 发起一次明确的格式修复请求。
3. 第二次失败则标记该席失败。
4. 阶段屏障失败，Session 不会静默推进。

## 投票

第一版默认允许自投，并在决策结果中记录 `self_vote_count`。是否禁止自投由 `VotePolicy` 控制，默认值是 `allow_self_vote = true`。

投票时策案以 `proposal_0`、`proposal_1`、`proposal_2` 映射到匿名策案，UI 可展示最终作者和来源，但投票请求阶段不依赖作者身份。
