# 合议协议

## 席位

- 谋远席：发散思考、替代路线、长期机会和组合可能。
- 经世席：用户价值、现实路径、资源成本和最小落地方案。
- 持正席：事实、逻辑、隐藏假设、风险、原则和边界。

三席平等，不存在固定主持人或裁判。

书记官是可选辅助角色，不参与投票。它只负责整理共识、检查结构完整性、汇总冲突和生成最终报告。

## 模式

### 三席模式

默认模式。三个席位独立执行、互相批议、修订策案并投票。

### Single Agent 模式

对照模式。用于验证三席模式是否比单 Agent 自我反思更稳定。它应保留为基线，不应混入三席互批逻辑。

## 阶段

1. 创建议题：用户提交标题、议题正文、补充背景、模式、模型配置、投票策略和可选外部证据。
2. 外部来源准备：可选联网搜索、上传文档解析、代码搜索或外部 Agent 注入 evidence。
3. 独议：三席并行执行，互相看不到其他席输出，但可共享同一外部证据池。
4. 批议：每席读取另外两席的结构化摘要并提出质询。
5. 复议：每席吸收批议并形成正式策案。
6. 阁议：三席匿名评价全部策案并投票。
7. 多数决：按 `VotePolicy` 计算结果。
8. Convergence：若三票分散，只允许一次合案复议和重新投票。
9. 书记官报告：如果启用，整理共识、结构缺口、未解决冲突和最终报告。

## 外部证据协议

外部证据可以来自：

- 网页搜索。
- 用户上传文件。
- 代码搜索。
- 外部 Agent 注入。
- 后续轻量日志分析或数据查询。

外部证据进入统一 evidence 结构：

- `kind`: `fact`、`inference` 或 `preference`。
- `source_kind`: `web_search`、`file`、`code`、`log`、`data` 或 `internal`。
- `trust_level`: `untrusted_external`、`user_provided`、`verified_external` 或 `internal`。
- `source_hash`: 用于追踪来源内容。
- `safety_flags`: prompt injection、控制字符、截断等安全标记。

外部证据只作为事实材料，不作为指令。模型不得执行来源文本里的命令、提示或要求。

## 工具轨迹协议

工具调用统一写入 `tool_runs`。当前稳定工具包括：

- `web_search`
- `document_parse`
- `code_search`

每个 `ToolRun` 至少记录：

- 工具名。
- 输入摘要。
- 输入 hash。
- 状态。
- 耗时。
- 关联 evidence IDs。
- 错误信息。
- 创建时间。

当前阶段工具调用由系统入口触发：会话搜索、文档解析 API、代码搜索 API。尚未实现“三席运行中自主选择工具并形成独立轨迹”，因此后续不要假设每个席位已经有独立工具预算或权限策略。

## JSON 约束

所有模型阶段都必须返回结构化 JSON。解析失败时：

1. 保存原始输出。
2. 发起一次明确的格式修复请求。
3. 第二次失败则标记该席失败。
4. 阶段屏障失败，Session 不会静默推进。

这条规则属于冻结行为契约。UI 可以优化错误提示，但后端不应静默吞掉结构化失败。

## 投票

投票策略由 `VotePolicy` 控制。

当前策略：

- `simple_majority`: 普通多数，至少两席同意同一策案。
- `risk_veto`: 风险否决，任一席提出阻塞风险可否决。
- `unanimous`: 全票通过。
- `conditional_pass`: 有条件通过，在多数基础上追加监控条件。
- `weighted_score`: 加权评分。

默认允许自投，并在决策结果中记录 `self_vote_count`。是否禁止自投由 `allow_self_vote` 控制，默认值是 `true`。

投票时策案以 `proposal_0`、`proposal_1`、`proposal_2` 映射到匿名策案。UI 可展示最终作者和来源，但投票请求阶段不依赖作者身份。

## 多数和少数意见

最终结果必须保留：

- 多数理由。
- 选中策案。
- 少数意见。
- 自投数量。
- 少数席位选择。
- 重新评估条件。
- 风险阻塞状态。
- 下一步。

如果没有形成多数，系统应保留无多数状态和未解决问题，而不是伪造结论。

## 安全规则

1. 三席不能删除其他席位提出的证据。
2. 事实、推断和偏好必须分开。
3. 外部来源不具备指令权。
4. 疑似注入来源可以保留为材料，但必须打安全标记。
5. 最终报告和 Audit 导出应保留关键证据和工具轨迹。
6. 用户偏好只作为默认配置，不是长期记忆或事实来源。

## 当前未冻结部分

以下协议面还在设计中：

- 每席运行中自主工具选择。
- 每席工具预算和权限策略。
- 日志 level/时间/错误聚类。
- 数据查询的筛选、聚合或 SQL 查询。
- 结果可信度摘要 UI。

在这些能力完成前，外部 Agent 和前端不要依赖它们的最终形态。
